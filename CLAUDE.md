## Development Workflow

Since this is a documentation-only repository, "development" means editing Markdown files.

1. Create a feature branch from `master`
2. Make changes (edit or add `.md` files)
3. Commit with a short, descriptive message
4. Open a pull request targeting `master`
5. Merge once reviewed
